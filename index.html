Format guide

How this practical paper works

You are expected to read stimulus material such as IAM policies, CLI output, user data scripts, subnet settings, automation artefacts, and platform health details before answering. The fastest way to score is to work from the artefact outward instead of guessing from memory.

Read in layers

  • Start with the scenario summary.
  • Scan artefacts for keywords like subnet, public IP, Deny, rate, device, healthy, or target CPU.
  • Then answer only from what the scenario proves.

Typical traps

  • Explicit Deny overrides Allow.
  • Private subnet plus null public IP means no public internet access.
  • User data normally runs only on first boot.
  • You cannot attach a snapshot directly; you create a volume from it first.
